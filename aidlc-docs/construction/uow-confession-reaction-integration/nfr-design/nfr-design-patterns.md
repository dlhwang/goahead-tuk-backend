# UOW-CONFESSION-REACTION-INTEGRATION NFR Design Patterns

<!-- markdownlint-disable MD060 -->

## 범위

이 단위는 Confession 단건 및 목록 읽기 결과에 허용된 세 Reaction
타입의 집계를 포함시키는 읽기 경로를 설계한다.

## 확정된 패턴

| 관심사 | 설계 패턴 | 구현 및 검증 책임 |
| ------ | --------- | ----------------- |
| 목록 집계 | 식별자 묶음 기반 단일 그룹 집계 질의 | 고해 식별자와 타입으로 group count를 한 번 읽어 N+1 집계를 금지한다. |
| 응답 완전성 | 응답 조립기의 zero-fill | 조회 결과에 없는 `PRAY`, `COMFORT`, `TOGETHER` 타입은 `0`으로 채운다. |
| 실패 경계 | 재시도 없는 fail-fast 읽기 | 집계 조회 실패를 가짜 `0`으로 바꾸지 않고 안전한 전체 오류로 전달한다. |
| 관측 | 안전한 실패 지표 및 구조화 로그 | 저장소 세부 정보나 식별자는 외부 응답과 로그에서 보호한다. |

## 성능 및 신뢰성 결정

- 목록 한 번마다 Reaction 집계 질의는 최대 한 번이다.
- Reaction 집계를 포함한 목록 읽기의 정상 운영 목표는
  `p95 <= 500 ms`로 유지한다.
- 요청 경로에서 집계 읽기 재시도를 하지 않아, 일시 장애 중
  중복 부하 확대와 오래 지연된 응답을 피한다.
- 운영 경보 및 로그 보존의 외부 증빙은 `UOW-SECURITY`와
  Infrastructure Design이 추적한다.

## PBT 및 검증 패턴

- 임의의 다중 고해 선택 set에 대해 집계 응답이 항상 세 타입을
  포함하는지 검증한다.
- 타입별 count가 입력 모델의 활성 선택 수와 일치하는지 검증한다.
- 서로 다른 고해의 선택이 다른 고해 집계로 섞이지 않는지 검증한다.
- 예제 기반 저장소 테스트는 목록 조회 시 단일 group query 사용과
  저장소 실패의 안전한 오류 전달을 검증한다.
