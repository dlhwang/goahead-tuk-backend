# UOW-CONFESSION-REACTION-INTEGRATION NFR Requirements

## 범위

이 단위는 기존 Confession 단건 및 목록 읽기 결과에 Reaction 타입별
활성 집계를 결합하는 품질 요구를 소유한다.

## 성능 및 확장성

- 고해 목록 조회는 목록 내 각 고해마다 별도 집계 질의를 실행하지
  않고, 한 번의 batch aggregate 조회를 사용해야 한다.
- 정상 운영 조건에서 Reaction 집계를 결합한 목록 응답의 서버 처리
  시간 목표는 `p95 <= 500 ms`다.
- 단건 조회는 `UOW-REACTION`의 단일 집계 제공 목표인
  `p95 <= 500 ms` 범위와 일관되어야 한다.

## 신뢰성 및 무결성

- 성공 응답은 항상 `PRAY`, `COMFORT`, `TOGETHER`의 세 타입을
  포함하고 미선택 타입은 `0`으로 반환한다.
- 조회 응답의 count는 Reaction aggregate port가 제공한 활성 선택
  집계와 동일해야 한다.
- 집계 획득 실패 시 고해 읽기 전체는 안전하게 실패하고, 실제 값이
  아닌 `0` 집계로 대체하지 않는다. (`SECURITY-13`, `SECURITY-15`)

## 보안 및 비노출

- 응답에는 기기 식별자, 작성자 판단 데이터나 개별 반응 행이
  포함되지 않는다. (`SECURITY-08`)
- 내부 집계 실패 원인 또는 저장소 세부 정보는 오류 응답으로
  노출하지 않는다. (`SECURITY-09`, `SECURITY-15`)

## 테스트 및 검증

- 예제 기반 테스트는 미선택 타입의 `0`, 복수 타입 집계, 단건 및
  목록 집계 실패 경로를 포함한다.
- PBT는 임의 선택 집합에 대한 세 타입 완전성, count 무결성, 단순
  모델 비교와 다중 고해 집계 분리를 검증한다.

## 후속 설계 입력

- NFR Design은 batch aggregate 조회 협력과 실패 경계 구현 패턴을
  구체화한다.
- Build and Test는 N+1 집계 질의 금지와 성능 목표 검증 증빙을
  남긴다.
