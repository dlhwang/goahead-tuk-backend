# UOW-REACTION Business Logic Model

## 단위 목적

익명 기기가 다른 사용자의 고해에 허용된 지지 반응을 선택하거나
해제하고, 활성 선택으로부터 타입별 집계를 제공한다.

## 선택 흐름

1. API 경계가 `confessionId`, `reactionType`, `X-Device-Id`를 받는다.
2. Reaction application은 식별자 및 허용 타입 검증을 통과한 입력만
   처리한다.
3. 대상 고해가 존재하는지 확인하고, 작성자의 익명 기기 식별이 요청
   기기와 같은지 판단한다.
4. 동일 기기이면 자기 고해 반응 금지 오류로 거부한다.
5. 다른 기기이면 `(confessionId, deviceId, reactionType)` 선택을
   활성 상태로 만든다.
6. 동일 선택이 이미 존재하면 저장 상태를 바꾸지 않고 성공한다.

## 해제 흐름

1. 선택 흐름과 같은 입력, 고해 존재 및 자기 반응 금지 규칙을
   적용한다.
2. 대상 선택이 존재하면 활성 선택 행을 제거한다.
3. 대상 선택이 없어도 상태 변경 없이 성공한다.
4. 선택 이력 보존은 이 MVP의 Reaction 기능 도메인 범위에 포함하지
   않는다.

## 집계 제공 흐름

1. aggregate port는 고해 ID 하나 또는 여러 개를 입력으로 받는다.
2. 활성 선택 행을 허용 타입별로 센다.
3. 허용 타입별 값으로 정규화할 수 있는 집계 값을 제공하며, 읽기
   통합 단위가 세 타입 완전 표현을 담당한다.
4. 집계는 개인별 선택 목록이나 기기 식별자를 반환하지 않는다.

## 오류 경로

- 대상 고해가 없으면 not-found 결과를 반환한다.
- 허용되지 않은 타입 또는 부적절한 식별 입력은 요청 오류로 거부한다.
- 작성 기기와 요청 기기가 같으면 자기 반응 금지 결과를 반환한다.
- 공통 throttling과 안전한 오류 표현은 `UOW-SECURITY`의 상세
  설계와 결합한다.

## Testable Properties (PBT-01)

- **Idempotence**: 임의의 유효한 타인 고해 선택에 동일 `PUT`을
  반복 적용한 관찰 가능한 선택 집합과 집계는 한 번 적용한 결과와
  같다.
- **Idempotence**: 임의의 유효한 타인 고해 선택에 동일 `DELETE`를
  반복 적용한 상태와 집계는 한 번 적용한 결과와 같다.
- **Invariant**: 모든 명령 순서 뒤에 동일
  `(confessionId, deviceId, reactionType)` 활성 선택은 최대 하나다.
- **Invariant**: 타입별 집계는 활성 선택 행 수와 항상 같다.
- **Stateful/Model-based**: 선택/해제 명령 시퀀스를 집합 모델에
  적용하고 각 단계 뒤 실제 저장 관찰값과 집계를 비교한다.
- **Round-trip**: API 입력 타입 문자열과 `ReactionType` 매핑이
  구현되면 허용 타입의 문자열 변환 왕복 속성을 코드 생성 계획에
  포함한다.
