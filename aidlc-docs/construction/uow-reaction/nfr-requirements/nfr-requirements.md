# UOW-REACTION NFR Requirements

## 범위

`UOW-REACTION`은 익명 반응 선택/해제 API, 별도 선택 저장 및
Reaction aggregate port의 품질 요구를 소유한다.

## 성능 및 확장성

- 정상 운영 조건에서 반응 선택, 해제 및 단일 고해 집계 제공의 서버
  처리 시간 목표는 `p95 <= 500 ms`다.
- 단일 애플리케이션 인스턴스는 지속 `20 requests/second`의 Reaction
  변이 요청을 오류 없이 처리하는 것을 초기 목표로 한다.
- 성능 시험은 허용된 rate-limit 범위 안에서 수행하며, 제한되는
  요청은 성공 처리량에 포함하지 않는다.
- NFR Design 답변에 따라 위 정량 목표는 MVP의 설계 목표로 유지하되,
  첫 전달의 차단 게이트로 사용하지 않는다. MVP 검증은 저장소 및
  통합 테스트와 제한된 동시성 테스트로 무결성 및 경쟁 안전성을
  증명하고, 정량 목표 검증은 배포 후 hardening 증빙으로 추적한다.

## 신뢰성 및 데이터 무결성

- 선택과 해제는 멱등하며 중복 활성 선택을 생성하지 않는다.
- `(confessionId, deviceId, reactionType)` 활성 선택 유일성은
  영속성 경계에서도 보장해야 한다.
- 집계는 활성 선택 행만으로 계산하며 `Confession` 타입별 카운트
  필드를 사용하지 않는다.
- 데이터베이스 처리 실패는 성공으로 위장하지 않고 안전한 오류
  경계로 전달한다.

## 보안 및 개인정보 보호

- 허용 타입, 고해 식별자와 기기 식별자 입력은 명시적인 형식과 길이
  제약을 가진다. (`SECURITY-05`)
- 자기 고해에 대한 반응 선택 및 해제는 `403 Forbidden`으로
  거부하며 작성자 관계 정보를 노출하지 않는다. (`SECURITY-08`)
- 원문 `X-Device-Id`는 응답이나 로그에 포함하지 않는다.
  (`SECURITY-03`, `SECURITY-08`)
- 공개 변이 API는 `UOW-SECURITY`가 정의하는 내부 throttling 정책의
  적용 대상이다. (`SECURITY-11`)
- 선택 유일성과 집계 파생은 데이터 무결성 검증 대상이다.
  (`SECURITY-13`)

## 테스트 및 검증

- 예제 기반 테스트는 성공 선택/해제, 반복 요청, 자기 고해 거부,
  없는 고해 및 입력 오류를 포함한다.
- PBT는 선택/해제 멱등성, 선택 유일성, 집계 불변식과 상태 시퀀스
  모델 비교를 다룬다.
- PBT 실행은 `Kotest Property Testing`과 CI seed 기록 정책을
  사용한다.

## 후속 설계 입력

- NFR Design은 트랜잭션 경쟁 처리와 성능 검증 접근을 구체화한다.
- Code Generation은 Reaction API 및 저장 경계를 구현하고 PBT와
  예제 기반 테스트를 함께 제공한다.
- Build and Test는 무결성 및 제한된 동시성 검증 결과를 기록하며,
  정량 성능 목표의 후속 검증 상태를 명시한다.
