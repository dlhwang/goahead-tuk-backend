# 고해 반응 MVP 사용자 스토리 평가

## 요청 분석

- **원 요청**: 익명 사용자가 고해에 댓글이나 자유 텍스트 없이 허용된
  긍정적 반응을 선택하고 해제할 수 있는 백엔드 기능을 추가한다.
- **사용자 영향**: 직접적이다. 고해를 읽는 사람이 새 상호작용 API를
  사용하고, 고해 조회 시 지지 반응 집계를 보게 된다.
- **복잡도**: 중간 이상이다. 기기별 멱등 선택 상태, 타입별 집계,
  `X-Device-Id` 보호, 공개 변이 API 남용 방지, property-based testing
  검증이 필요하다.
- **이해관계자**: 익명 고해 열람자, 고해 작성자, API 소비자, 백엔드
  유지보수자.

## 적용 기준

- [x] 높은 우선순위: 사용자가 직접 조작하는 새 기능이다.
- [x] 높은 우선순위: `PUT` 및 `DELETE` 고객 대상 API 계약이 추가된다.
- [x] 높은 우선순위: 선택, 반복 선택, 해제, 반복 해제, 집계 조회라는
  복수 사용자 시나리오가 있다.
- [x] 중간 우선순위: 별도 반응 선택 데이터와 집계가 사용자 데이터 및
  조회 응답에 영향을 준다.
- [x] 중간 우선순위: 익명 식별 데이터 보호와 공개 API 남용 방지가
  수용 기준과 테스트에 연결되어야 한다.

## 결정

**User Stories 실행**: Yes

**판단 근거**: 고해 반응은 단순 내부 리팩터링이 아니라 익명
열람자에게 새 행동을 제공하고 고해 작성자가 확인하는 집계를 바꾸는
기능이다. 스토리를 작성하면 멱등 선택 집합 계약과 개인정보 비노출,
허용 타입 제한, 남용 방지 기대를 구현 및 검증 단계까지 추적할 수
있다.

## 기대 결과

- 익명 열람자의 긍정 반응 선택 및 해제 여정을 수용 기준으로 표현한다.
- 고해 작성자 또는 열람자가 집계를 확인하는 결과를 명시한다.
- 기기별 중복 방지, 비노출, 허용 타입 제한과 공개 API 남용 방지를
  테스트 가능한 조건으로 연결한다.
- 기존 `US-1 Anonymous Device Writes A Confession` 스토리는 보존하고,
  Reaction 범위만 추가하는 근거를 남긴다.
