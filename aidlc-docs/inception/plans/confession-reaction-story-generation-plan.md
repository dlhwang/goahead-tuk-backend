# 고해 반응 MVP 사용자 스토리 생성 계획

<!-- markdownlint-disable MD053 -->

## 목표

[confession-reaction-requirements.md](../requirements/confession-reaction-requirements.md)의
승인된 요구사항을 바탕으로, 익명 고해 반응 선택과 조회 결과를 사용자
중심 스토리와 수용 기준으로 추가한다. 기존 익명 고해 작성 스토리는
그대로 유지한다.

## 승인 대기 중인 스토리 접근 방식

- **페르소나 범위**: 새 관점은 `익명 고해 열람자` 하나로 추가한다.
  집계 확인은 반응을 남기는 열람자가 확인할 수 있는 결과로 다룬다.
- **스토리 구성**: 기능 기반으로 반응 선택/해제 스토리와 반응 집계
  조회 스토리를 분리한다.
- **보안 수용 기준**: 정상적인 익명 반응 사용은 과도하게 차단하지
  않되 반복적인 남용 요청은 제한된다는 기대를 선택/해제 스토리에
  포함한다.
- **기존 산출물 처리**: 기존 익명 고해 작성자 페르소나와 작성
  스토리는 보존하고 Reaction 항목만 추가한다.

## 스토리 구성 선택지

- **사용자 여정 기반**: 선택, 해제, 집계 확인을 사용자가 수행하는
  순서 중심으로 분리한다. 행동별 수용 기준은 명확하지만 API 상태
  계약이 여러 스토리에 반복될 수 있다.
- **기능 기반**: 기기별 반응 선택 상태 관리 한 스토리와 집계 확인
  스토리로 구성한다. API와 조회 결과의 책임 경계가 분명하다.
- **단일 기능 스토리**: 반응 선택/해제/집계를 한 스토리의 수용 기준으로
  묶는다. 문서는 간결하지만 보안 및 멱등 시나리오가 길어진다.
